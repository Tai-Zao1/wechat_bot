# runtime 目录说明

`runtime/` 放的是运行过程中共享的状态能力。

它和 `services/` 的区别是：

- `services/` 更偏业务流程
- `runtime/` 更偏跨进程、跨任务、跨窗口状态协调

当前主要有两块：

1. `scheduler.py`
   微信 UI 执行权、任务接管、跨进程锁、运行时心跳。
2. `self_profile.py`
   当前登录账号的本地资料缓存与刷新。

如果你在排查“为什么任务互相抢窗口”或者“为什么新实例会接管旧实例”，先看 `scheduler.py`。

如果你在排查“为什么当前账号昵称 / wxid 没刷新”或者“为什么本机缓存没有写入”，先看 `self_profile.py`。

如果你在写新代码：

- 业务流程逻辑不要堆到 `runtime/`
- 只有真正跨任务共享、需要本机持久化或运行时协调的能力，才放这里
